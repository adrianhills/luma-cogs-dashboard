The margin reality: Ray (own) clears a healthy ~56% (marginal lens); third-party API generations run thin (~8–11%) — you're largely passing the vendor's fee through and can't engineer it away. API is ~18% of generations but ~51% of compute spend: the case for steering volume onto Ray as it matures, and pricing API-served clips to protect margin.

What this is. A mock decision surface built for the Luma TPM technical session — to make the COGS conversation concrete rather than abstract. Every number is illustrative and chosen to be order-of-magnitude plausible; none are Luma's real figures. The structure — credits as the customer-facing unit, Ray vs. third-party API split, marginal vs. fully-loaded margin, allocation-vs-utilization gap — mirrors how Luma actually prices and serves today.

Pricing structure grounded in Luma's public pricing (Plus $30 / 10k credits · Pro $90 / 40k · Ultra $300 / 150k; video ≈ 4–280 credits/sec by model, resolution & audio; Ray in-house + Veo/Kling/Seedance via API; Dream Machine, Luma Agents, and Agents API are separate products with separate credit wallets).
